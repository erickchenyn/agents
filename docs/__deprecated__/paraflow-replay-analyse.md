## 核心功能

- 使用 moxt-cli 的 project:replay 工具下载并解析回放数据（在项目根目录下执行 `pnpm exec tsx cli/src/index.ts project:replay <replay-file-url>`）
- 使用 moxt-cli 的 project:clone 工具克隆项目仓库（在项目根目录下执行 `pnpm exec tsx cli/src/index.ts project:clone <project-id>`）
- 结合结构化的回放数据和 Git 提交历史，提取关键信息，重建时间线，生成分析报告

## 时间线中的关键信息

- **用户会话**:
  - 用户与 Agent 对话，每一轮会话 Pipeline 包含若干次对话 ChatTurn
  - 会话事件位于回放数据的 `chatTurns` 中
- **Agent 提交代码**:
  - Agent 在和用户对话的过程中会修改代码并提交到 Git 仓库。Agent 产生的提交信息包含 snapshot 关键词
  - Agent 提交信息不在客户端回放数据中，只能通过 Git 仓库的提交历史获得
- **客户端同步**:
  - 客户端事件数据位于回放数据的 `funcCalls` 中，其中 Git 相关的事件名为 `recordGitWorkflow`，apply 的事件名为 `applyRemoteDiffsToDocument`
  - 客户端按固定间隔 pull 尝试获取 Git 仓库的最新代码，检测到变更后 apply 到编辑器并让画布区展示
  - 若客户端产生用户编辑操作，会尝试提交并 push 到 Git 仓库。客户端产生的提交信息包含 editor 关键词
  - 如遇到 pull/push 不能 fast-forward 的情况，会在本地处理冲突然后再次尝试 pull/push，此时会有处理冲突的事件记录

**关键节点**:

- 会话开始/结束
- Agent 提交代码
- 客户端 pull & apply
- 客户端 commit & push

**关键时长**：

- 会话时长
- 客户端 pull 时长，尤其是存在 apply 之前的那次 pull
- 客户端 push 时长

## 生成分析报告

- 包含关键时间点、关键时长的完整时间线
- 发现关键时间点之间的顺序、关键时长数据有明显不合理的地方
- 分析用户体验及可能存在的问题

**用户视角下的预期**：

- 会话期间，Agent 每次提交代码后，用户未能及时观察到画布区更新，不符合预期
- 会话结束后，用户长时间仍未观察到画布区更新，严重不符合预期
