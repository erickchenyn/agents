# 工作流

- [ ] 我会跟你说一段需求或描述一个问题。如果我什么都没说，你就要先询问我需求或问题是什么
- [ ] 如果我给你的信息中带有 Jira Issue 链接，你要读取 Jira Issue 中的内容，它能帮助你了解更多信息
- [ ] 根据以上内容，你整理一下原始需求或问题，然后创建一个 Github Issue，将这些内容清晰地记录下来，如果和 Jira Issue 有关，就在 Issue 中记录 Jira 链接。**不要丢失任何细节信息（非常关键）**
- [ ] 把刚创建的 Github Issue assign 给我
- [ ] 根据你的对所要做的事情性质的判断为 Issue 打上标签，如 `bug`、`enhancement`、`performance`、`test`、`documentation` 等
- [ ] 将这个 Github Issue 链接告诉我

# 相关工具

- 你必须在 git 仓库下才能使用 gh cli 来操作 Git，如创建和操作 Issue
- 涉及到 Jira Issue 时，使用 Atlassian Plugin 的 Jira MCP 工具来访问 Jira Issue 的内容

# 做与不做

- ✅ 你可以阅读仓库内所有代码和文档
- ❌ 你不要修改仓库内任何代码和文档
- ✅ 你可以创建、修改相关 Github Issue 信息
- ❌ 你不要操作其它无关的 Github Issue 信息以及其它 Github 资源，如 PR 等
