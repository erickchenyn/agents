---
name: workspace-remove
description: 移除指定的 git worktree，并将其 Claude 配置合并回主工作区以保持记忆延续
---

## 工作流

1. **检查工作区环境**
   - 检查是否在 git 仓库中
   - **检查是否在主工作区**：
     - 使用 `git worktree list --porcelain` 获取所有工作区信息
     - 第一个工作区即为主工作区，检查其路径是否与当前工作目录 `pwd` 一致
     - 如果不在主工作区，显示详细的错误提示：
     - "❌ 此 skill 必须在项目主工作区中执行"
     - "当前目录：[当前路径]"
     - "主工作区：[主工作区路径]"
     - "请使用 `cd [主工作区路径] && cc` 切换到主工作区后重试"
     - 主工作区路径示例：如果主工作区在 `/home/wukong/.claude`，则必须在此目录下执行
   - 只有在主工作区中才能继续后续操作
   - 获取当前项目的根目录路径

2. **列出所有 worktree**
   - 使用 `git worktree list --porcelain` 获取所有 worktree
   - 过滤掉主工作区（通过检查是否为 bare 或当前目录）
   - 展示所有可移除的 worktree 列表，包括路径、分支名、状态

3. **用户选择要移除的 worktree**
   - 提供多选界面让用户选择要移除的 worktree
   - 支持选择单个、多个或全部非主工作区
   - 显示每个 worktree 的基本信息（路径、分支名等）

4. **对选中的 worktree 进行安全检查**
   - **工作区状态检查**：检查工作区是否干净（无未提交的更改）
   - **分支状态检查**：检查分支是否有未推送的提交
   - **PR 状态检查**：使用 GitHub CLI 检查对应分支的 PR 状态
     - 检查是否已创建 PR
     - 检查 PR 是否已合并或关闭
     - 如果 PR 仍开放，警告用户可能有进行中的工作
   - **远程分支检查**：检查远程分支是否已被删除
   - 为每个选中的 worktree 生成安全报告
   - **智能确认策略**：
     - 如果所有检查都通过（安全删除），标记为可直接删除
     - 如果发现任何风险项，展示详细风险报告并询问用户是否强制删除

5. **备份 Claude 配置**
   - 对于每个待移除的 worktree，检查是否存在 `.claude/settings.local.json` 文件
   - 如果存在，读取其内容并与主工作区的 `.claude/settings.local.json` 进行智能合并
   - 合并策略：
     - 数组类型：去重后合并
     - 对象类型：深度合并，子 worktree 的配置优先级更高
     - 基本类型：保留最新的值
   - 将合并后的配置写回主工作区

6. **执行删除操作**
   - 对每个选中的 worktree 执行 `git worktree remove <path>`
   - 如果删除失败，提供强制删除选项

7. **验证和报告**
   - 验证 worktree 已成功移除
   - 显示最终的 worktree 列表
   - 报告配置合并的详细信息（如果有）
   - 汇总删除操作的结果

## 使用要求

**⚠️ 重要提醒**：此 skill 必须在项目的主工作区中执行，否则无法正确列出和管理所有 worktree。

### 正确的使用方式
```bash
# 1. 确保在主工作区中
cd /path/to/main/workspace
cc  # 启动 Claude Code

# 2. 然后使用 workspace-remove skill
/workspace-remove
```

### 错误处理
如果在非主工作区中执行，skill 会自动检测并提示正确的切换命令。

## 安全性
- 在删除前会进行多重安全检查（未提交更改、未推送提交、PR 状态等）
- 自动备份并合并 Claude 配置文件，确保不丢失个性化设置
- 支持批量删除和单个删除操作
