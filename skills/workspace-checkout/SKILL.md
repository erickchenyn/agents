---

## 执行流程

1. **调用脚本执行切换操作**
   - 使用 Bash 工具调用 `workspace-checkout.sh` 脚本（位于 `~/.claude/skills/workspace/` 目录）
   - 传递用户提供的分支名或 PR ID 作为参数
   - 传递其他可选参数（如 `-u` 用户名、`-e` 邮箱、`-d` 预览模式等）
   - 捕获脚本的输出结果和执行状态

2. **解析脚本输出**
   - 从脚本输出中识别操作类型（创建新 worktree 或切换到现有 worktree）
   - 提取目标工作区路径和分支名称
   - 获取脚本执行的详细步骤信息

3. **验证切换结果**
   - 使用 `git worktree list` 确认目标 worktree 存在且状态正常
   - 检查目标工作区目录是否存在且可访问
   - 验证分支状态是否正确：
     - 如果是新创建的 worktree，确认分支已正确创建
     - 如果是现有 worktree，确认代码已更新到最新版本
   - 检查 git 配置是否正确（用户名和邮箱）
   - 确认代码生成是否成功完成

4. **验证 PR/分支解析**
   - 如果输入是 PR ID，验证是否成功解析为对应的分支名
   - 使用 GitHub CLI 确认分支或 PR 的状态信息
   - 检查远程分支是否存在且可访问

5. **报告执行结果**
   - 成功情况下，向用户报告：
     - 工作区路径和分支信息
     - 操作类型（新建或切换）
     - 切换到工作区的命令提示
     - 确认所有初始化步骤完成状态
   - 失败情况下，报告具体错误原因：
     - 分支或 PR 不存在
     - 工作区创建失败
     - 网络或权限问题
     - 提供相应的解决建议

6. **提供后续操作建议**
   - 指导用户如何切换到目标工作区
   - 建议验证工作区环境是否符合预期
   - 如有冲突或问题，提供故障排除步骤
