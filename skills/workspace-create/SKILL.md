---

## 执行流程

1. **调用脚本执行创建操作**
   - 使用 Bash 工具调用 `workspace-create.sh` 脚本（位于 `~/.claude/skills/workspace/` 目录）
   - 传递用户提供的参数（如 `-u` 用户名、`-e` 邮箱等）
   - 捕获脚本的输出结果，包括成功消息和可能的错误信息

2. **解析脚本输出**
   - 从脚本输出中提取新创建的工作区路径
   - 提取新创建的分支名称
   - 识别脚本执行状态（成功/失败）

3. **验证创建结果**
   - 使用 `git worktree list` 确认新的 worktree 已成功创建
   - 检查新工作区目录是否存在且可访问
   - 验证新分支是否已创建并正确关联到 worktree
   - 检查 git 配置是否正确设置（用户名和邮箱）
   - 确认 `.claude/settings.local.json` 是否已正确拷贝到新工作区

4. **报告执行结果**
   - 如果创建成功，向用户报告：
     - 新工作区的路径和分支名
     - 切换到新工作区的命令提示
     - 确认所有初始化步骤已完成
   - 如果创建失败，报告具体错误原因并提供建议解决方案

5. **提供后续操作建议**
   - 提示用户如何切换到新创建的工作区
   - 建议用户验证工作区环境是否正常
   - 如有必要，提供清理失败创建的工作区的方法
